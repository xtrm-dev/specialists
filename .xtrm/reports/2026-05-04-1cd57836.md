## Previous xt Report Context To Carry Forward

The previous report `.xtrm/reports/2026-05-04-2b52300a.md` is essential context. Carry these points forward:

1. **Manifest/tool resolver became canonical.** `PiSessionOptions` now carries `specialistName` / `specialistPermissions`; legacy hardcoded permission-tier arrays and `SPECIALISTS_USE_RESOLVER` rollout flag were removed from `src/pi/session.ts`. Serena still enforces native tool denial at runtime; resolver overrides mainly provide explicit policy, token savings, and restore safety.
2. **Cat A vs Cat B distribution model.** Cat A runtime assets — specialists, mandatory rules, catalog, nodes — should resolve live from the specialists package after project/user tiers. Cat B filesystem assets — skills and hooks — are owned by xtrm-tools under `.xtrm/skills/default` and `.xtrm/hooks/default`.
3. **Drift and source verification.** Use `sp doctor --check-drift`, `sp prune-stale-defaults --dry-run`, and `sp config show <name> --resolved --from-source` when reviewing resolver/catalog behavior inside a worktree. Global dist can be stale relative to worktree source.
4. **Epic/merge recovery.** `failed -> abandoned` recovery shipped; `sp ps` now hides terminal epics by default and has filters (`--running`, `--bead`, `--since`, `--mine`, `--include-terminal`). Dirty-tree or dependency-cycle failures in `sp epic merge` may still require careful manual `git merge --no-ff` of successful chain heads, followed by `sp epic abandon --force` bookkeeping.
5. **Dispatch safety shipped.** `sp stop` sibling guard, epic state writeback/self-heal, and dispatch race claim guard all landed. The claim guard uses atomic SQLite transaction in `claimJobStart`; if startup failures recur, check this area first.
6. **Known noisy failure.** `tests/unit/cli/run.test.ts` had 9 pre-existing failures on pristine master in the previous session; issue `unitAI-51a85` tracks it. Do not let executors chase those as regressions unless revalidated.
7. **Payload/startup metric work exists.** Several commits and issues around startup payload metrics are present. `unitAI-xqwgx` should use those surfaces before redesigning skill injection.
