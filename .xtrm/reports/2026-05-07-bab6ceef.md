## Code Changes

### sp serve operational logging

- `src/cli/serve.ts` now parses `--log-level off|info|debug` with default `info`, emits structured JSON operational logs, truncates errors through `shortLogError`, and keeps a stable request trace id for paths that fail before script-runner metadata exists.
- `/v1/generate` logs normal script-runner results using `result.meta` where available and logs malformed JSON, invalid request shape, shutdown, queue timeout, and unexpected internal errors without logging request bodies.
- `src/index.ts` serve help now advertises `--log-level off|info|debug`.
- `tests/integration/sp-serve.test.ts` covers default success logging, `--log-level off` suppression, malformed request logging/no body leak, and existing observability DB behavior.
- `dist/index.js` and serve declaration artifacts were refreshed after the source merge.

### Local dev container naming
