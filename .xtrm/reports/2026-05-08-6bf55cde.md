## Code Changes

### Adopter-contract surface (server + Python client)

- `src/specialist/script-runner.ts`: added `expected_output_keys` validation path. New helper `collectRequiredOutputKeys(spec)` unions `execution.expected_output_keys` (any `response_format`) with `prompt.output_schema.required` (only when `response_format === 'json'`). The post-attempt success block now JSON-parses + key-checks whenever the union is non-empty, returning `invalid_json` with the missing-key message on failure.
- `src/specialist/schema.ts`: `ExecutionSchema.expected_output_keys: string[]` field added with explanatory comment.
- `clients/python/specialists_client.py`: stdlib-only ~170 LOC client. Closed `error_type` enum mirrors the server taxonomy 1:1 (now includes `template_field_misuse`, `prompt_too_large`, `output_too_large`) plus a caller-side `transport` for HTTP/socket failures.
- `clients/python/{pyproject.toml,README.md,tests/test_smoke.py}`: minimal package + smoke tests gated by `SPECIALISTS_SERVICE_URL`. Live-verified against darth-feedor's running container at `172.18.0.27:8000`.

- `config/specialists/test-runner.specialist.json` (v2.0.0): manifest-detect pre-script, project-language-aware system prompt + task_template. Tags scrubbed.
