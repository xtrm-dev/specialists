## Code Changes

### `sp ps -f` rendering

- `src/cli/ps.ts` now treats follow mode as a mini-TUI for TTY output: enter alternate screen (`?1049h`), hide cursor, render at cursor home with erase-down, skip unchanged frames, and restore cursor/screen on `SIGINT`, `SIGTERM`, normal `exit`, or pipe `EPIPE`.
- Non-TTY follow output now strips ANSI escape sequences and appends plain snapshots separated by blank lines. This preserves useful `sp ps -f > log.txt` behavior while avoiding terminal-control garbage in logs.
- The implementation intentionally leaves the existing `renderBuffered()` wrapper in place to avoid rewriting the dashboard renderer; only the output transport changed.

### Help, dist, and generated artifacts

- `src/index.ts` help text for `--follow/-f` was updated to document the new semantics: TTY repaints in-place, pipes append ANSI-free snapshots.
- `dist/index.js` and `dist/types/cli/ps.d.ts.map` were rebuilt with `bun run build` so the packaged CLI matches source.
- `CHANGELOG.md` gained an `[Unreleased]` Fixed entry for `unitAI-fqo38` because this is observable CLI behavior.
