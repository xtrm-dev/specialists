# Session Report — 2026-06-02

## Summary

The AgentOps telemetry bridge for xtrm/specialists moved from design language to a working pre-substrate implementation: runtime events now have a canonical `xtrm.forensic.v1` envelope, durable `specialist_forensic_events` persistence, deterministic redaction before storage/output, additive `forensic_event` payloads on `sp feed --json` and `sp log --json`, a new `sp forensic <job-id> --json` drill-down command, a low-cardinality Prometheus/OpenMetrics projection via `sp metrics --prometheus`, and a read-only `GET /metrics` surface in `sp serve`. The bridge was validated first with unit/integration/build checks and then with a real specialist job (`ddc421`, `explorer`, `done`): `sp forensic ddc421 --json` returned 68 forensic events across `job`, `model`, `turn`, `tool`, and `git`; `sp feed ddc421 --json` included `forensic_event`; and `sp metrics --prometheus --since 24h` projected explorer aggregates without `job_id`, `chain_id`, or `participant_id` labels. The documentation now separates forensic evidence from aggregate metrics, Gitboard has a companion PR exposing structured feed events, and three follow-up beads capture the next product/design layer: Gitboard UI rendering, substrate design revision, and a complete AgentOps event catalog.

## Code Changes

### Forensic runtime and persistence

`src/specialist/forensic-events.ts` now builds canonical `xtrm.forensic.v1` envelopes from legacy timeline events, classifies event families/names, carries participant identity (`participant_kind`, `participant_role`, optional model/provider), stores correlation fields (`job_id`, `tool_call_id`, bead/chain context), and applies deterministic redaction before persistence/output. `src/specialist/observability-sqlite.ts` added schema version 12 with `specialist_forensic_events`, indexes by job/sequence/time/family/participant, dual-write at the event persistence choke point, and query helpers for `sp forensic`.

### CLI surfaces

`src/cli/forensic.ts`, `src/index.ts`, and `src/cli/help.ts` added the new `sp forensic [job-id] --json` query command. `sp feed --json` and `sp log --json` now include additive `forensic_event` payloads without removing legacy fields. This was intentionally additive so Gitboard and other consumers can migrate gradually.

### Prometheus projection and HTTP exporter

`src/specialist/prometheus-projection.ts` implements table-derived Prometheus/OpenMetrics rendering and validation. `src/cli/metrics.ts` and `src/cli/serve.ts` expose it as `sp metrics --prometheus` and `GET /metrics`. The metric catalog covers job state, job totals, duration/wait histograms, queue depth, process/worktree gauges, active runtime, turns, context usage, tool calls, LLM tokens, and projection timestamp while preserving the label discipline: low-cardinality identity labels only.

### Tests and build artifacts

Unit tests cover forensic envelope creation/redaction, SQLite schema/persistence/query, Prometheus rendering/label validation, command help, feed/log compatibility, and `sp serve` `/metrics`. `.github/workflows/telemetry-contract.yml` adds parser validation in CI. `dist/` and `dist/types/` were rebuilt to ship the new CLI/API surfaces.

### Cross-repo Gitboard work

External PR `https://github.com/Jaggerxtrm/gitboard/pull/38` adds `GET /api/specialists/jobs/:job_id/feed-events`, which shells out to `sp feed <jobId> --json`, parses NDJSON, and returns structured events. It leaves the terminal/text feed path intact. This branch was developed in a clean worktree to avoid the user's dirty Gitboard checkout, then pushed and cleaned up locally.
