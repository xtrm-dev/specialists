# Session Report — 2026-05-05

## Summary

Cut release `v3.12.0` and shipped two follow-on epics that materially advanced the v3 specialist orchestration design and uncovered a load-bearing observability bug. Epic `unitAI-m8744` enriched `sp list --full` to surface worktree behavior, chain-position, median runtime, and role-specific mandatory rules — making the live registry the source of truth that v3 SKILL.md defers to. The v3 skill itself was expanded from a slim 285 lines to a 562-line canonical orchestration manual covering bead-as-prompt rewrites, dependency-linking forms (`bd dep add`/`--blocks`/`relate`/`--parent`/`discovered-from:`), per-bead-type CONTRACT shapes, three dependency-graph diagrams, dedicated `## Code-sanity` and `## Security-auditor` subsections with handoff semantics, mini-flows for under-promoted specialists (planner/debugger/overthinker/sync-docs/test-runner/researcher), an `## Adjacent xt commands` cross-ref subsection, opt-in XML guidance for evidence-heavy fields, a `REFERENCES (load on demand)` field, and explicit chain-of-thought role placement. Epic `unitAI-tmspo` shipped two new mandatory rules (`per-turn-handoff-schema` for JSON-validated end-of-turn handoffs across 12 specialists; `bead-id-verbatim` to stop the dropped-dot bd-id typo affecting gpt-5.4-mini) plus the matching v3 SKILL.md guidelines. Bug `unitAI-kdl4n` (P1) fixed a payload over-counting defect at `runner.ts:1184` where each attached mandatory rule was attributed the FULL combined block, inflating reported `payload_kb` by N×; the fix introduces per-set sections in `formatMandatoryRulesBlock`. A live KPI audit then revealed `payload_breakdown` events also UNDER-count (skill bodies are eagerly injected via `skills.paths` but only path/description labels are recorded — true cost shows only in `token_trajectory_json[0].input_tokens`), leading to Recipe 7 corrections and a new Recipe 8 in `/using-kpi` and a complete reframe of bead `unitAI-xqwgx`.



Late session closed the xtrm-unblocked specialists-side distribution work. `unitAI-tsnwh.5` rewrote `update-specialists` into the operator wrapper over `xt doctor --cwd` and `xt update --root/--repo --apply`, with explicit Category A/Category B language, dry-run/confirmation flow, post-update doctor check, single/multi-repo handling, and graceful fallback when `xt` is missing. `unitAI-o4khi` then landed the consolidated documentation pass for the canonical-live model: new `docs/installation.md`, distribution notes in authoring/manifest/skills/hooks/CLI docs, specialists-creator guidance for canonical rule/skill references, CHANGELOG entries, and OpenWolf notes. Both tracks were reviewed PASS and pushed (`da5d1edd`, `b65aba14`). The important operational caveat is that xtrm-tools already has byte-identical mirrored `update-specialists` in `.xtrm/skills/default/update-specialists/SKILL.md`, but those xtrm-tools mirror changes remain uncommitted there; publishing xtrm-tools is what makes the skill available to users without a specialists repo clone.


Final close-out published `@jaggerxtrm/specialists@3.13.0` after the earlier runtime-update flow exposed two release-critical packaged-runtime regressions: bundled `sp doctor`/`sp status` crashed because `version-check` resolved `../../package.json` from installed `dist/index.js`, and `security-auditor` shipped machine-local optional skill paths under `/home/dawid/projects/xtrm-tools`. The fix made package-version resolution layout-tolerant (`../package.json` for dist, `../../package.json` for source), rewrote optional security skill paths to repo-relative `.xtrm/skills/optional/...`, included `update-specialists` v2.1 in the package, rebuilt `dist/`, tagged/pushed `v3.13.0`, and published npm latest. Validation covered `bun run lint`, version-check unit tests, `npm pack --dry-run`, local `npm install -g .`, `sp doctor --check-drift`, `sp prune-stale-defaults --dry-run`, `sp clean --dry-run`, `sp list --compact`, and `sp --version`; npm latest now reports `3.13.0`. A separate later `f61032a5` serve fix landed on `master` for `--offline` propagation to script-runner Pi invocation.
