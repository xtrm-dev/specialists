## Code Changes

### Script runner subprocess contract

- `src/specialist/script-runner.ts` is the central hardening surface. It now builds isolated Pi argv with no context files, no prompt templates, no themes, no tools, offline mode, and either trusted repeated `--skill` args or `--no-skills`.
- Rendered prompts are written to child stdin and closed; prompt bytes are checked before spawn against `execution.prompt_limit_bytes` / `SPECIALISTS_SCRIPT_PROMPT_LIMIT_BYTES`.
- `prompt.system` is forwarded as Pi `--system-prompt` as a full override, not appended text.
- JSON specialists receive a minimal output contract derived from `output_schema`/required keys.
- Local script hooks fail closed in script/service mode.
- Trusted skills use path-boundary root validation and audit entries include whether the source came from `skills.paths` or `prompt.skill_inherit`.

### CLI/service behavior

- `src/cli/script.ts` and `src/cli/serve.ts` now honor exact observability DB file semantics for `--db-path`.
- `sp serve` passes `projectDir` into `runScriptSpecialist()` so the Pi subprocess cwd matches the service project.
- `sp serve` gained optional Pi readiness canary modes (`off`, `warn`, `require`) with explicit failure reasons while preserving `/healthz`.
- The service path still uses one Pi subprocess per in-flight request; SDK work is intentionally deferred.

### Tests and build artifacts

- Unit/integration tests cover stdin prompt transport, cwd wiring, prompt-size preflight, JSON contract injection, trust root boundaries, local script rejection, exact DB path behavior, and readiness canary modes.
- `dist/index.js`, `dist/lib.js`, and generated type files were rebuilt for the hardening commits before merge.
- Final post-merge validation passed: 68 Vitest tests, 36 Bun SQLite tests, `npm run lint`, and `npm run build`.
