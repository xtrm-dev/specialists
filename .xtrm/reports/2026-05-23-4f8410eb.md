# Session Report ‚Äî 2026-05-23

## Summary

Reviewer crash issue `unitAI-6x6p6` was reproduced, diagnosed, fixed, reviewed, merged, and documented. The important discovery is that the exact `17408` cache-token fingerprint was not causal: a fresh heavy-review reproducer (`cada28`) died with the same stale `running -- dead` behavior at `cache_read_tokens=14848`, while lighter reviewer probes succeeded at overlapping/higher cache counts. The durable root cause was a supervisor lifecycle gap: if a specialist process dies during a heavy bash validation burst before `run_complete`, `Supervisor.readStatus()` computed `is_dead` but left persisted status as `running`; `sp result` therefore kept saying ‚Äústill running‚Äù until manual stop. The merged fix (`4f8410eb`) adds read-time dead-PID reconciliation for `starting|running` jobs, emits a terminal `run_complete(ERROR)` event, guards malformed timestamps, respects file-output-off mode, and adds a focused regression test. `CHANGELOG.md` now records the user-facing fix under `[Unreleased]`. Cleanup ended with no active specialist jobs and no session-created worktrees; several pre-existing global Claude/Serena/tmux/tmp artifacts remain and are called out below as not opened by this session.

## Code Changes

### Supervisor dead-job reconciliation

- `src/specialist/supervisor.ts` now routes SQLite and file-fallback `readStatus()` results through `reconcileDeadStatus(id, status)`.
- Dead `starting|running` jobs are converted to `status: "error"`, get `error: "Process crashed or was killed"`, and emit `run_complete(ERROR)` with `exit_reason: "crashed"` so `sp ps` / `sp result` no longer loop on stale non-terminal state.
- The helper preserves existing liveness semantics: `waiting` is not newly terminalized, `EPERM` PID checks remain treated as alive via existing `isPidAlive`, invalid `started_at_ms` skips recovery instead of emitting malformed events, and file-output-off mode no longer writes stray `events.jsonl` entries.

### Regression coverage

- `tests/unit/specialist/supervisor-read-status-dead.test.ts` covers SQLite recovery, file-output-off behavior, and malformed `started_at_ms` skip behavior.
