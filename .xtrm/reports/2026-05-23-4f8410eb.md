## Code Changes

### Wave A — Supervisor dead-job reconciliation

- `src/specialist/supervisor.ts` now routes SQLite and file-fallback `readStatus()` results through `reconcileDeadStatus(id, status)`.
- Dead `starting|running` jobs are converted to `status: "error"`, get `error: "Process crashed or was killed"`, and emit `run_complete(ERROR)` with `exit_reason: "crashed"` so `sp ps` / `sp result` no longer loop on stale non-terminal state.
- The helper preserves existing liveness semantics: `waiting` is not newly terminalized, `EPERM` PID checks remain treated as alive via existing `isPidAlive`, invalid `started_at_ms` skips recovery instead of emitting malformed events, and file-output-off mode no longer writes stray `events.jsonl` entries.
- `tests/unit/specialist/supervisor-read-status-dead.test.ts` covers SQLite recovery, file-output-off behavior, and malformed `started_at_ms` skip behavior.
