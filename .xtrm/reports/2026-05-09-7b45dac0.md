## Code Changes

### `src/specialist/supervisor.ts`

- **`gitnexusHasEmbeddings(cwd)` (NEW, exported)** — reads `.gitnexus/meta.json`, returns `true` only when `stats.embeddings` is a positive number; falls back to `false` on missing file, missing key, zero count, or malformed JSON.
- **`startDetachedGitnexusAnalyze(cwd)` (modified)** — now passes `--embeddings` to `npx gitnexus analyze` when `gitnexusHasEmbeddings(cwd)` is true, otherwise keeps the bare invocation.
- **`triggerGitnexusAnalyzeIfNeeded(sha, source)` (NEW closure-local helper)** — gated on `isGitnexusAnalyzeRequired(runOptions.permissionRequired)` (MEDIUM/HIGH only); deduped via `lastGitnexusAnalyzedSha` (a closure-local `let`); spawns the detached analyze and emits a `gitnexus_analyze_started` (or `_start_failed`) META timeline event with `backend: 'checkpoint' | 'terminal'`. Updates `lastGitnexusAnalyzedSha` on success when sha is known.
- **`applyAutoCommitCheckpoint(...)` (modified)** — after the success-path `auto_commit success` event, calls `triggerGitnexusAnalyzeIfNeeded(autoCommitResult.sha, 'checkpoint')`.
- **Terminal-path inline gitnexus block (replaced)** — the previous 16-line `if (isGitnexusAnalyzeRequired(finalResult.permissionRequired)) { try { startDetachedGitnexusAnalyze(...); ... } catch {...} }` block at the post-finalResult site is now a single `triggerGitnexusAnalyzeIfNeeded(statusSnapshot.last_auto_commit_sha, 'terminal')` call. Dedup ensures the same final-commit sha never analyzes twice.
