# Session Report — 2026-05-06

## Summary

Fixed a long-standing bug where `script-runner.ts` ignored `spec.specialist.prompt.system`, causing pi to inherit its default coding-agent system prompt when spawning non-coding specialists. The root cause was that `runSingleAttempt` had no pathway to forward the system field — it only built args for `--mode`, `--no-session`, `--no-extensions`, `--no-tools`, `--offline`, `--model`, and `--thinking`. With instruction-tuned models (glm-5, qwen-thinking, dashscope qwen3-coder-plus), pi's default system prompt contains explicit instructions to read `docs/prompt-templates.md` when it sees the word "template" — which appears in every rendered `task_template`. This caused those models to emit `<tool_use>` blocks looking for pi's own internal documentation instead of returning structured output. The fix adds `--system-prompt <value>` (full override, not append) when `spec.specialist.prompt.system` is non-empty, matching the intent of the field and allowing darth-feedor callers to drop the `f"{system}\n\n{prompt}"` workaround added in ddea367. Two new tests were added asserting presence/absence of the flag. Three pre-existing TypeScript errors in the test file (missing `afterEach` import from vitest, incomplete loader mock casts) were also fixed in the same pass.

## Code Changes

### Key modifications

**`src/specialist/script-runner.ts`** — `runSingleAttempt` gains an optional `systemPrompt: string | undefined` parameter. When truthy, `['--system-prompt', systemPrompt]` is pushed onto the pi args array after `--thinking` (if present) and before the positional user prompt. In `runScriptSpecialist`, `spec.specialist.prompt.system || undefined` is resolved once per model loop iteration and forwarded. The choice of `--system-prompt` (full override) over `--append-system-prompt` is deliberate: for non-coding consumers like darth-feedor, pi's coding-agent baseline is the bug, not a feature, so full override is correct. Specs that want pi's coding baseline simply leave `prompt.system` empty.

**`tests/unit/specialist/script-runner.test.ts`** — Two new describe blocks added under `runScriptSpecialist system prompt forwarding`: one asserts `--system-prompt` and its value appear in `spawnMock.mock.calls[0][1]` when a spec has `prompt.system` set; one asserts the flag is absent when the spec has no system field. Pre-existing TS errors fixed: `afterEach` added to vitest import, all `makeLoader()` and partial options literals cast with `as never` where needed by the `ScriptRunnerOptions` type constraint.
