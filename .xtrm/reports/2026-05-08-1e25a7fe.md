# Session Report — 2026-05-08

## Summary

Combined chain-lifecycle and epic-simplification redesign landed end-to-end on master in 9 functional commits + 4 doc commits, closing the three deadlock loops captured in the prior `unitAI-9z8dp` investigation. Headline change: epic state is now derived live from chain readiness (not a persisted state machine), per-chain `sp merge` works for any PASS chain regardless of sibling-epic state, reviewer PASS auto-finalizes the keep-alive executor through the supervisor's canonical path, and a new `sp finalize <any-chain-job-id>` cascade-closes every waiting keep-alive member of a chain when auto-finalize misses. ## Code Changes

### Source

- **`src/specialist/supervisor.ts`** — Added PASS auto-finalize hook for keep-alive executors (shouldAutoFinalizeKeepAlive); `finalizeWaitingJob()` canonical terminator reusing updateJobStatus(); exposed `listChainJobIds(chainId)` delegating to sqlite for the cascade fix; updated TERMINAL/PASS verdict regexes to match markdown-bold.
- **`src/cli/finalize.ts` (NEW)** — Operator command that cascades closure across an entire chain on reviewer PASS. Accepts any chain member (executor/reviewer/debugger), looks up chain via chain_id, scans for PASS reviewer, calls `supervisor.finalizeWaitingJob()` on every waiting member.
- **`src/specialist/jobRegistry.ts`** — `finalize()` alias for the canonical waiting-close path.
- **`src/specialist/epic-readiness.ts`** — Rewired evaluateEpicReadinessSummary off live prep/chain data; legacy `failed` rows recover to merge_ready when live chains pass; failed chains now appear in blockers + human summary; verdict regex updated for markdown-bold.
- **`src/specialist/epic-lifecycle.ts`** — Relaxed evaluateEpicMergeReadiness to pure live-chain terminality check.
- **`src/cli/epic.ts`** — Removed `sp epic resolve` subcommand entirely (not a no-op shim); validateEpicMergeReadiness only blocks `merged`/`abandoned`, not persisted `failed`.
- **`src/cli/merge.ts`** — checkEpicUnresolvedGuard inverted-gate removed; consults derived chain readiness instead.
- **`src/cli/resume.ts`, `src/cli/stop.ts`, `src/cli/ps.ts`** — ps header uses epicStateLabel(readiness_state) (derived); resume refuses finalized jobs cleanly; stop preserves sibling-live-job guard.
- **`src/specialist/runner.ts`** — Added SHELL_BUILTINS allowlist before commandExists() check; fixes `unitAI-0vaas`.
- **`src/index.ts`** — Wired `finalize` subcommand and updated help.
